Part 1 of 2

The mode

The value that appears most often in a data set — the most popular value, like the tallest bar on a column graph.

Mode = the value that appears most frequently. “Mode” rhymes with “most” — a handy way to remember.

Find the mode — tap a number

Tap any value to light up every time it appears and count it. Then reveal the mode.

Tap a number to start counting…

How to find the mode

  1. Count how many times each value appears (tally as you go).
  2. The value with the highest count is the mode.
  3. If two values tie for most — you have two modes.
TeacherWork it together. Ask students to tally before revealing. Encourage crossing out / tally marks as they count.

Mode from a graph

On a column graph the mode is simply the tallest bar — no need to count every data point.

TeacherAsk: “Which sport should the school buy more equipment for, to match what most students enjoy?” Soccer = 14, the tallest bar.

Can there be more than one mode?

Yes! When two values tie for most frequent, the data is bimodal — two peaks.

2  5  5  7  8  5  2  9  2  6
TeacherBoth 2 and 5 appear 3 times → two modes. Real example: survey students and teachers about age → two peaks.

Part 2 of 2

Distribution shape & skewness

A distribution shows how data is spread — where most values cluster, and whether the data leans to one side. The peak is the mode; the tail tells us the direction of the skew.

➡️ Positively skewed

Peak on the left, tail stretches right. Most values are low; a few large ones drag the tail out.

  • Mode: 0–1 hours (tallest bar, far left)
  • Real life: most students watch little TV; a few watch a lot
Tip: “Positive” → tail points to the positive (right) end.

⬅️ Negatively skewed

Peak on the right, tail stretches left. Most values are high; a few low ones drag the tail out.

  • Mode: 12–13 points (tallest bar, far right)
  • Real life: most students aced an easy quiz; a few struggled
Tip: “Negative” → tail points to the lower (left) end.

Scenario 2 — Pocket money ($)

5, 5, 5, 5, 10, 10, 15, 20, 30, 50
  1. What is the mode?
  2. What is the range? (max − min)
  3. If you drew a column graph, which way would the tail stretch?
  4. Is this positively or negatively skewed?
  5. Why might a few students receive $50? What does that tell us?
✓ Answers
  • Scenario 1: (a) 9–10  ·  (b) negatively skewed  ·  (c) most students scored well; only a few got low scores.
  • Scenario 2: (a) $5  ·  (b) $45  ·  (c) right  ·  (d) positively skewed  ·  (e) open discussion — those few are outliers / high earners pulling the tail out to the right.

🔴 Red Extended challenge Reason deeply · full independence

Problem 1 — Two classes, two distributions

Class A: 2, 2, 2, 4, 5, 6, 7, 8, 9, 10

Class B: 2, 4, 5, 6, 7, 8, 9, 10, 10, 10
  1. What is the mode of Class A? Of Class B?
  2. Which class has positively skewed data? Which has negatively skewed data?
  3. Which class performed better overall? Use the distribution shape as evidence.

Problem 2 — Design your own data set

Create a data set of 10 values that is positively skewed and has a mode of 3. Write the values, sketch the graph, and explain why it is positively skewed.

★ Extension: Think of a real-life situation where you’d expect negatively skewed data. Explain why the data skews that way, and what the mode would represent in that context.

✓ Answers & guidance
  • Problem 1: Class A mode = 2 (positively skewed — most scored low). Class B mode = 10 (negatively skewed — most scored high). Class B performed better: its scores cluster at the high end, so the bulk of students did well.
  • Problem 2: Open-ended. One example: 3, 3, 3, 3, 3, 4, 5, 7, 10, 15 — peak (mode 3) on the left, tail of larger values stretching right.
  • Extension: Accept any reasoned example — e.g. driving-test scores when most people pass; age at retirement; an easy quiz. The mode represents the most common outcome in that context.
TeacherLook for evidence-based reasoning that links the shape to performance, not just the mode value.
